# NOAA current stations, bundled offline

*Validated design. 2026-07-23. Owner: Bryan.*

The web app serves currents CHS-online-only today (IWLS → IndexedDB). US current
stations — Deception Pass, Admiralty, San Juan Channel, Rosario — are absent entirely,
even though their harmonic constituents are public domain and `slackwater-engine`
already computes them offline from the same data. An app named Slackwater that cannot
answer "when is slack?" in US water without signal is backwards. This closes that gap,
before the discovery map (`2026-07-23-discovery-map-design.md`).

## 1. Data: a second build-time bundle

`scripts/build-currents.mjs`, sibling to `build-stations.mjs`:

- Source is the `currents.json` **shipped inside** `@sailingnaturali/current-stations`
  (new devDependency) — no network at build time, same posture as `@neaps/tide-database`.
- Filters, all load-bearing:
  - Salish Sea bbox (reuse the same `[-125.5, 47.0, -122.0, 50.5]`).
  - **Harmonic stations only.** Subordinate stations are offsets from a reference and
    need the engine's reduction math; deferred (`ponytail:` ceiling comment in the
    script names the upgrade path).
  - **Primary bin only.** NOAA publishes multiple depth bins per station; the bundle
    marks the primary. One station, one prediction.
  - Zero-amplitude constituents dropped.
- Output `src/data/currents.json`, gitignored, regenerated by `dev`/`build`/`test`.
- Fails the build if the filtered bundle is empty (mirror `build-stations.mjs`).
- Licence test asserting every bundled station's source is NOAA (public domain), same
  shape as the existing CHS licence test.

Per station: id, name, position, timezone, mean flood/ebb directions, constituents
(name/amplitude/phase, velocity major axis).

## 2. Prediction: the harmonic sum the app already runs

`@neaps/tide-predictor` fed current constituents yields **signed major-axis velocity
in knots** instead of height — no datum offset, positive is flood. This is exactly how
`slackwater-engine` does it (validated: PUG1741 max 9.7 min / 0.055 kn vs NOAA's own
predictions).

- Timeline: velocity sampled across the day, same cadence as the tide chart.
- Events: extremes are max flood / max ebb; **zero crossings are slack**. The predictor
  gives extremes; slack detection is a small sign-change scan over the sampled timeline.
- Set bearing at time `t` = mean flood direction when v > 0, mean ebb direction when
  v < 0 (matches the engine).

New module `src/noaaCurrents.ts` holding bundle load + prediction + events, shaped to
emit the same `CurrentState` the CHS path produces so downstream components don't fork.

## 3. Identity: station-corrections, as always

NOAA current stations get registry entries in `@sailingnaturali/station-corrections` —
that package is the workspace's source of truth for station identity, and the web's
slugs/aliases/gazetteer already flow from it. Cross-repo change, sequenced first:
registry entries land and publish, then this app bumps the dependency.

Registry `station:` key carries the NOAA id (`noaa/PUG1741`-style), `kind: current`.
Naming follows the registry's existing membership rules.

## 4. UI: no new views

- Stations join list / search / nearby as current stations, same treatment as gates.
- Detail renders with the existing `CurrentChart` — with a real timeline, since unlike
  a derived gate these have one. `EventList` shows slack / max flood / max ebb rows.
- **No offline status chrome**: prediction is local, so the CHS `loading`/`offline`
  states simply never occur for these stations. They are also excluded from the
  offline-sync download accounting — there is nothing to download.
- URLs: `/tide/<slug>` (or the gate-style route the resolver assigns) — whatever the
  registry slug is, exactly like bundled NOAA tide stations. Raw id fallback
  (`noaa/PUG1741`) resolves and redirects, matching existing behaviour.

## 5. Error handling

- Build-time: empty bundle or non-NOAA licence fails CI.
- Runtime: none new — no network involved. A station with degenerate constituents is
  excluded at build time, not guarded at runtime.

## 6. Testing

- **Golden fixture**: capture NOAA's own predictions with `current-stations golden`
  (PUG1741 as in the engine; one Salish pass fixture checked in). Assert slack/max
  times within 15 min and speeds within 0.1 kn — engine-report tolerances, rounded up.
- Unit: slack zero-crossing detection (sign change, no-crossing day, double slack),
  bbox/primary-bin/licence filters, `CurrentState` shape parity with the CHS adapter.
- Existing suites (search, nearby, EventList) pick the new stations up via fixtures.

## 7. Deferred

- **Subordinate stations** — needs the engine's reduction math ported; the harmonic
  set covers the passes that matter first.
- Multiple depth bins per station.
- CHS↔NOAA duplicate-neighbourhood arbitration (a US station and a BC gate near a
  border pass both showing in Nearby is acceptable; both are honest).
